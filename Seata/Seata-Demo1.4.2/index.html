seata 1.4.2版本 代码示例

https://gitee.com/aacopy/seata-learn/tree/1.4.2/

版本信息

  • seata 版本 1.4.2
  • mysql 版本 8.0
  • nacos 版本 2.0.2
  • SpringCloud 版本 2020.0.4 ，AlibabaCloud 版本 2021.1

代码说明

四个服务：

  • Account 账户服务
  • Order 订单服务
  • Storage 库存服务
  • Business 业务服务

业务服务下单，调用库存服务，扣减库存，调用订单服务，下单，订单服务中再调用账户服务，扣款

启动seata服务端

下载服务端

下载地址：http://seata.io/zh-cn/blog/download.html 点击1.4.2版本的binary

创建数据库脚本

  1. 创建数据库seata
  2. 执行建表SQL脚本，脚本地址：https://github.com/seata/seata/blob/v1.4.2/script/server/db/mysql.sql

注意：

  • seata在创建事务组的时候，如果在客户端中不指定配置seata.tx-service-group，则默认以：spring.application.name值+”-seata-service-group“拼接后的字符串作为分组名，例如：seata-business-seata-service-group。而在sql脚本中global_table表的transaction_service_group字段的长度默认为32，是不够的，会报以下错误：
    • 解决方案1：把global_table表的transaction_service_group字段长度加大，改为64
    • 解决方案2：在客户端配置文件中，配置seata.tx-service-group的值，使长度小于等于32

nacos中添加seata配置文件

  • nacos中新增配置

    • Data ID 填写 seata.properties
    • Group 填写 SEATA_GROUP
    • 配置格式 选择 Properties
    • 配置内容 配置内容如下，以下为最小配置项，其他都有默认配置

  • 其他配置项内容可以参考官方提供的配置文件内容，获取地址：https://github.com/seata/seata/blob/v1.4.2/script/config-center/config.txt

  • 注意事项：

    • store.db.driverClassName配置项文档中给的值是com.mysql.jdbc.Driver，需要改为com.mysql.cj.jdbc.Driver，不然启动会报以下错误
    • store.mode配置项数据库模式，配置项db一定要小写，一开始参考最新分支的注释，写成了大写，结果启动会报以下错误
    • store.db.datasource=druid必须要配置，不配置会报如下错误
    • store.db.dbType=mysql必须要配置，不配置会报如下错误

修改seata配置文件

  • 打开根目录conf文件夹
  • 编辑修改registry.conf配置文件
  • 修改registry和config的type为nacos，并修改nacos的相关配置

启动seata服务端

  • 进入seata根目录中的bin目录下，执行命令（linux）

  • 注意：

    • 因为seata服务端在启动时，可能无法找到本机正确的ip，导致客户端连接时，会报错，并且查看nacos注册中心，服务的ip也不对，所以最好在启动的时候，加上-h 192.168.80.128
    • 非docker安装的服务，注意需要关闭服务器防火墙，不然客户端无法访问到seata服务端

  • 查看nohup日志没有报错，并且nacos服务管理，服务列表中有seata-server服务，即表示启动成功
