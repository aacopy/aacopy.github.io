1. SpringCloudAlibaba

SpringCloudAlibaba提供了微服务开发的一站式解决方案

2. 架构演进

2.1 单体架构

优点：便于集成，易于测试，部署，对小型项目友好

缺点：启动慢，依赖庞大，CI/CD复杂度高，升级重构成本大

2.2 分布式架构

  • SOA： 面向服务的架构，服务间通过相互依赖提供功能，服务之前通过网络调用

  • 微服务： 将一个大的单体应用进行细粒度的服务化拆分，每个服务各自打包部署，服务间通过网络调用

优点：易开发，理解和维护，独立部署和启动，微服务间是松耦合的，可以通过不同的语言开发，一个组件故障不会导致整个系统宕机

缺点：存在分布式事务问题，服务治理

2.3 分布式架构考虑的因素

  • 服务编排调度
    • 应对不对增长的服务实例和容器规模，Kubernetes
  • 服务注册发现
    • 解决众多微服务之间复杂的调用关系，Consul，Zookeeper，Nacos，Eureka
  • 服务状态，集群管理
    • 解决庞大规模集群的服务和机器运行状态，Kubernetes
    • Skywalking可以解决复杂的分布式链路追踪，监控各个服务的状态
  • 容器数据存储
    • 解决容器存储是临时的问题，Kubernetes Persistent volume
  • 网络
    • 服务间的通讯通过RPC的方式，依赖网络，需要确认网络的可用性及带宽，优化网络IO的开销
  • 容器隔离技术
    • 多容器在同一个主机上运行，管理进程与进程之间的访问权限，Flannel，Weaveworks，Calico
  • 监控，审核，日志
    • Loggly，Fluentd，日志条目，Datadog，ELK

2.4 构建微服务的目标

  • 降低维护成本
  • 提高部署效率
  • 提高系统弹性
  • 监测服务状态

2.5 构建微服务的关键的

  • 可伸缩性
    • 在系统扩展过程中，通过较少的改动就能实现整体系统处理能力的提升
    • 侧重系统的水平扩展，通过加机器实现分布式计算
  • 可用性
    • 通常使用多少个9来衡量系统可用性，比如99.99%即4个9表示1年中系统有一个小时不可用
    • 服务可用性 = （服务周期总分钟数 - 服务不可用分钟数）/ 服务周期总分钟数 * 100%
    • 一个服务周期为一个自然月
    • 通过故障服务自动隔离，调用降级实现高可用
  • 弹性能力
    • 即从某些类型的故障中恢复并保持功能的能力
    • 很大程度上取决于微服务的通讯弹性
  • 可扩展性
    • 水平扩展：添加现有节点相同功能的新节点，在重新分配负载
    • 垂直扩展：增加CPU，内存，磁盘，网络接口等提高机器的性能
  • 高内聚，低耦合
    • 需要以面向对象思想设计系统
  • 服务治理能力
    • 管理众多应用服务
  • 故障隔离
    • 调用方要确保自己具有隔离故障的能力，避免被调服务不可用
  • DevOps持续交付
    • DEV（开发人员），OPS（运维人员）
    • DevOps强调的是团队间可以通过自动化工具来进行协作沟通贸易完成软件生命周期管理，从而更快，更频繁的交付更稳定的版本

微服务架构核心组件

  • 网关：路由转发及过滤器，将Restful API以统一域名的方式暴露给app或者浏览器
  • 服务发现与注册：调用方和被调用方的信息维护
  • 配置中心：管理配置
  • 链路追踪：分析调用链路耗时
  • 负载均衡：分发流量到多个节点
  • 熔断：保护自己和被调用方

常见微服务的解决方案

  • SpringCloudAlibaba
    • https://spring.io/projects/spring-cloud-alibaba
    • 组件：
      • 通信方式：http restful (open-feign)，Dubbo
      • 注册中心：nacos
      • 配置中心：nacos
      • 断路器：sentinal
      • 网关：gateway
      • 链路追踪：sleuth+zipkin
      • 分布式事务：Seata
      • 消息中间件：RocketMQ
  • SpringCloud
    • https://spring.io/projects/spring-cloud
    • 组件：
      • 通信方式：http restful
      • 注册中心：eruka
      • 配置中心：config
      • 断路器：hystrix
      • 网关：zuul/gateway
      • 链路追踪：sleuth+zipkin
  • dubbo
    • https://dubbo.apache.org/zh/
    • 组件：
      • 通信方式：rpc
      • 注册中心：zookeper/redis/nacos
      • 配置中心：diamond，nacos

SpringCloudAlibaba与SpringCloud的区别和选择

  • SpringCloud很多组件是基于第三方整合的，目前很多已经不更新或者闭源了，比如zuul，eureka，hystrix
  • SpringCloudAlibaba提供一站式微服务解决方案，已经和SpringCloud进行了整合，组件相互支持
