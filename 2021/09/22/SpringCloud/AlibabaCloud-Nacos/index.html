什么是nacos

官方地址：https://nacos.io/zh-cn/index.html

nacos服务端(v1.4.2)

下载路径

在官网的首页中点对应的版本进入github，在最下面有下载入口

单机模式启动

  1. cd到nacos的bin目录下
  2. sh startup.sh -m standalone
  3. 查看进程是否已启动 ps -ef|grep nacos
  4. 查看启动日志 tail -300f ../logs/start.out
  5. 浏览器访问地址：http://127.0.0.1:8848/nacos/
  6. 登录账号密码，默认 nacos/nacos

在nacos中添加配置项

打开nacos地址，选择配置管理，选择配置列表，点击右侧➕

Data ID命名规则：${prefix}-${spring.profiles.active}.${file-extension}

  • prefix 默认为 spring.application.name 的值，也可以通过配置项spring.cloud.nacos.config.prefix来配置。

  • spring.profiles.active 即为当前环境对应的 profile，注意：当 spring.profiles.active 为空时，对应的连接符 -

    也将不存在，dataId 的拼接格式变成 ${prefix}.${file-extension}

  • file-exetension 为配置内容的数据格式，可以通过配置项 spring.cloud.nacos.config.file-extension 来配置。目前只支持 properties (默认)和 yaml 类型。
