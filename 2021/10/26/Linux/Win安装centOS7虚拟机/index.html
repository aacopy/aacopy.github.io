Win安装centOS7虚拟机

发表于2021-10-26|更新于2022-06-06|Linux
|阅读量:

安装VMware Workstation

  • 下载地址：https://www.vmware.com/cn/products/workstation-pro/workstation-pro-evaluation.html
  • 安装
  • 激活：16版本的序列号：ZF3R0-FHED2-M80TY-8QYGC-NPKYF

下载centOS7镜像

  • 下载地址：http://mirrors.aliyun.com/centos/7/isos/x86_64/
  • 下载标准安装版本DVD.iso，如：CentOS-7-x86_64-DVD-2009.iso

新建centOS7虚拟机

  1. 打开VMware，选择创建新的虚拟机
  2. 选择稍后安装操作系统
  3. 选择linux 和 centos7 64
  4. 点击完成
  5. 在虚拟机页面点击编辑虚拟机设置
  6. 可以移除USB，声卡，打印机等无用的设备
  7. 选择CD/DVD
  8. 使用ISO映像文件，选择iso文件
  9. 点击确定
  10. 点击启动虚拟机
  11. 语言选择英语的就可以 continue
  12. date&time选择亚洲，选择down
  13. 选择INSTALLATION DESTINATION，直接点down
  14. 选择NETWORK & HOST NAME 设置为ON
  15. 点击begin installation
  16. 设置ROOT PASSWORD 如123456. 点击down
  17. 等待加载完后点击reboot

设置虚拟机网络

三种网络模式

Bridged桥接模式

  • 默认使用VMnet0，不提供DHCP服务（DHCP服务是指由服务器控制的一段IP地址范围，当客户机登录服务器时会自动获取服务器分配的IP地址与子网掩码）
  • 虚拟机与外部主机需要在同一个网段上，与局域网的其它机器没有区别。
  • 可以与局域网内其它主机通信，可以与外部网络通信
  • 容易与局域网其他主机引起ip地址冲突

Host-Only仅主机模式

  • 默认使用VMnet1，提供DHCP服务
  • 虚拟机可以和物理主机互相访问，但虚拟机无法访问外部网络

NAT模式

  • 默认使用VMnet8，提供DHCP服务
  • 虚拟机可以和物理主机互相访问，可访问外部网络
  • 局域网内其它机器访问不了

​ 由于需要将闲置windows电脑作为虚拟机使用，并对其他局域网内的电脑可见，所以选用桥接模式，

右键服务->设置->网络适配器->选择桥接模式，并勾选上复制物理网络连接状态->确定

重启网卡：systemctl restart network.service

查看ip命令：ip addr

问题

问题1、启动报错：此主机支持 Intel VT-x，但 Intel VT-x 处于禁用状态。

（1）重启系统，进入BIOS（ThinkPad系列按F1）

（2）进入BIOS，选择Configuration选项，选择Intel Virtual Technology并回车

（3）若无VT选项或不可更改

（4）将光标移动至Enabled处，并回车确定

（5）最后按F10热键保存并退出即可开启VT功能

ping不通的时候，看下防火墙是不是关了

如果需要添加过个虚拟机，最简单的方式就是在已经添加好的虚拟机上右键->管理->克隆，选择全量

mac可以使用终端ssh@root:ip登录，传输文件可以下载免费的filezilla(https://www.filezilla.cn/download/client),选择sftp登录

设置固定IP和连接互联网

  • 管理员模式启动VMware

  • 编辑 -> 虚拟网络编辑器

  • 设置网络

    • 选择VMnet8
    • 点击还原默认设置
    • 取消勾选使用本地DHCP
    • 设置子网ip，改为需要设置的ip段
    • 点击NAT设置

    • 修改网关IP，最后一个要设置为2

    • 点击确定

  • 设置linux虚拟机

    • vim /etc/sysconfig/network-scripts/ifcfg-ens32
    • 如下几项是需要添加或者修改的

  • 重启网络服务

设置时间同步

在虚拟机右键 –> 设置 –> 选项 –> VMware Tools –> 勾选将客户机与主机同步
